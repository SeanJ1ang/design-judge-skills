# `design-award-search` 技能

[English](README_EN.md)

从设计奖官方来源检索并核验同类别获奖作品，以高精度案例支持竞品研究、设计基准和视觉比较。

## 适合用它做什么

- 查找与用户、功能、技术、干预机制、形态、场景、系统或视觉语言相关的获奖作品。
- 核验案例是否真实获奖、是否属于官方页面、是否有可检查的官方图像。
- 建立少量高相关性案例，而不是堆积泛化搜索结果。

## 典型请求

- “为这个康复训练产品寻找同类别 iF 和 Red Dot 获奖案例。”
- “找外观语言相近的获奖产品，并说明可观察到的相似点。”
- “核验这几个案例是否来自官方获奖页面。”

## 你需要提供

- 设计图片、核心功能、用户与使用场景。
- 希望关注的奖项、年份或相关性维度；没有时可由技能规划。

## 工作方式

先确定主功能类别，再按八个相关性维度建立查询；只保留通过官方域名、项目页面和视觉证据核验的候选案例，并明确主要关系、次要关系与相关性等级。

## 产出

- 高精度获奖案例表和官方链接。
- 每个案例的相关维度、证据状态和选择理由。
- 可观察的视觉比较记录与无法核验的缺口。

## 内置参考

- 共享功能分类体系与官方来源注册表。
- 查询构建、官方 URL 校验和视觉证据检查脚本。
- 检索政策与八维相关性定义。

## 边界

- 不评价或优化用户设计，不选择最适合的奖项。
- 不把搜索摘要或第三方合集作为最终证据。
- 不持久化或再分发官方图片、原文和私有案例数据。

## 相关技能

后续评价使用 `design-evaluation`；选择申报目标使用 `design-award-match`。
