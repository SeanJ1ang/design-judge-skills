# `design-evaluation` 技能

[English](README_EN.md)

使用透明、证据化量表评价单个设计或经用户确认成熟度的批量作品，并把设计质量、展示质量和证据置信度分开报告。

## 适合用它做什么

- 诊断设计本体与展示表达的优缺点。
- 对学生概念或成熟作品进行可重复评分。
- 在相同成熟度轨道内批量评价、隔离失败案例并形成 shortlist。
- 使用 iF、iF Student、Red Dot 或 IDEA 的观察性获奖样本作为描述性背景。

## 典型请求

- “按学生概念轨道评价这套展板，指出 Critical 问题。”
- “批量评价这些作品，但不要把学生概念和成熟产品混排。”
- “结合 iF 产品设计背景解释各维度表现。”

## 你需要提供

- 设计图片、展板、说明、研究或验证材料。
- 由用户确认的 `student_concept` 或 `mature_work` 成熟度轨道。
- 可选的目标奖项与类别。

## 工作方式

先分类并建立证据清单，再加载核心量表、成熟度配置、行业覆盖和可选奖项背景；确定性脚本负责权重、边界和批量排序。

## 产出

- 设计本体与展示表达的分维度评分。
- 每项评分的证据状态与理由。
- Critical 风险、证据置信度和优先改进方向。
- 批量任务中的分轨 shortlist 与失败隔离报告。

## 内置参考

内置 22,125 条 iF、iF Student、Red Dot 与 IDEA 获奖或入围作品的聚合观察背景；它只提供描述性上下文，不改变核心分数，也不代表官方权重。

## 边界

- 不模拟官方评委，不预测获奖概率。
- 不在不同成熟度轨道之间直接排名。
- 不自动重设计作品，也不把缺失视频或图片证据当作已验证事实。

## 相关技能

用 `design-award-search` 查找案例，用 `design-award-match` 选择申报目标。
