# `design-information-prep` 技能

[English](README_EN.md)

从用户授权的设计材料提取可追溯事实，建立可复用项目档案，并按目标奖项的具体字段准备申报文字。

## 适合用它做什么

- 从 brief、展板、报告、说明书、专利或既有文案整理项目事实。
- 发现申报前缺失的信息和相互冲突的说法。
- 按奖项字段、语言和字数限制起草或改写申报文字。

## 典型请求

- “根据附件准备 IDEA 申报文字，先列缺失事实。”
- “把现有中文项目说明适配为 iF 英文字段。”
- “建立一份以后可以复用到多个奖项的 project dossier。”

## 你需要提供

- 用户有权使用的项目附件和既有事实材料。
- 已确认的目标奖项、路径、类别与申报周期。

## 工作方式

先核验目标字段规格，再把事实、推断和缺失项分开写入项目档案；每个输出字段记录使用的事实 ID，并通过确定性脚本校验必填项、长度和来源。

## 产出

- 可复用、带来源记录的项目档案。
- 缺失事实和待确认问题清单。
- 按目标表单字段组织的中英文草稿。
- 长度、必填项和来源校验结果。

## 内置参考

包含多个支持奖项的字段规格、项目档案与输出 JSON Schema、证据政策和输出模板。

## 边界

- 不根据获奖案例文案虚构项目事实。
- 不选择目标奖项，不替代最终提交文件检查。
- 对动态字段和当届限制必须重新核验官方来源。

## 相关技能

先用 `design-award-match` 确定目标，完成文字后用 `design-submission-check` 终检。
