# `design-submission-check` 技能

[English](README_EN.md)

针对明确的奖项、路径、类别和申报周期，检查提交包的完整性、技术规格、跨材料一致性、权利风险与最终就绪度。

## 适合用它做什么

- 提交前检查展板、图片、视频、文字和证明材料是否齐全。
- 核对当届文件格式、尺寸、时长、匿名性和必填要求。
- 检查不同材料中的名称、数字、声明和事实是否一致。
- 输出可执行的 go / conditional go / no-go 判断。

## 典型请求

- “按今年 iF Student 规则检查这个提交包。”
- “找出展板、说明书和申报文字之间的事实冲突。”
- “判断当前是否可以提交，并按严重性列出问题。”

## 你需要提供

- 明确的奖项、项目路径、类别和周期。
- 实际提交文件或结构化 manifest。
- 用户提供的官方规则文件；没有时由技能实时核验。

## 工作方式

先建立带来源和核验日期的 requirement ledger，再检查材料清单和技术约束，随后执行跨材料事实、权利与披露检查，最后按 Blocker、Important 和 Advisory 汇总。

## 产出

- `Ready`、`Conditional go` 或 `No-go` 结论。
- 逐项规则证据、通过状态和问题严重性。
- 必须修复、建议修复与人工确认清单。

## 内置参考

包含检查框架、规则证据政策、一致性与权利清单、结构化示例和确定性 manifest 检查脚本。

## 边界

- 不选择奖项，不重新评价设计本体，也不重写整套申报文案。
- 不提供法律许可意见；权利问题需要相应专业人员确认。
- 无法核验动态官方规则时不得返回 `Ready`。

## 相关技能

申报文字由 `design-information-prep` 准备；目标奖项由 `design-award-match` 确定。
