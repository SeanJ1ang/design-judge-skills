# `design-award-pipeline` 技能

[English](README_EN.md)

用于判断当前应进入哪个 Design Judge 模块，并在多个阶段之间维护简洁、可恢复的交接记录。

## 适合用它做什么

- 从设计材料开始规划完整参奖路径。
- 不确定应该先检索、评价、匹配还是准备申报文字。
- 跨多个阶段推进，并保留已确认事实、决策与阻塞项。

## 典型请求

- “帮我制定这个学生概念的完整参奖计划。”
- “我只有展板和说明书，下一步应该做什么？”
- “继续上次的奖项匹配结果，进入申报文字准备。”

## 你需要提供

- 当前已有材料和希望完成的目标。
- 已确定的成熟度、目标奖项或时间限制；没有时可留空。

## 工作方式

它选择最小充分路线，调用对应专业 Skill，并使用结构化 handoff 保存事实、决策、产物路径、未解决问题和下一阶段。

## 产出

- 当前推荐路线及理由。
- 已完成阶段与关键决策摘要。
- 阻塞项、待核验规则和下一步行动。

## 边界

- 不替代五个专业 Skill 的规则与输出。
- 不强制执行完整流水线；单一任务只路由到一个 Skill。
- 不预测获奖概率，不把观察性获奖样本当作官方评审权重。
