# `design-award-match` 技能

[English](README_EN.md)

在 11 个已配置设计奖项目中匹配奖项、项目路径和申报类别，先执行结构性资格门控，再比较公开标准与项目证据。

## 适合用它做什么

- 判断设计适合投哪个奖、哪个项目或类别。
- 比较多个奖项的适配度、资格风险和申报优先级。
- 为学生概念、成熟产品或专业项目筛选合理路径。

## 典型请求

- “比较 iF Student、Red Dot Concept、DIA 和 James Dyson。”
- “这个成熟产品适合 Red Dot Product 还是 IDEA？”
- “检查申请人身份、上市状态和时间条件是否构成硬性风险。”

## 你需要提供

- 项目状态、主要功能、申请人身份、时间信息和已有证据。
- 希望比较的奖项；未指定时可从支持列表建立候选集。

## 工作方式

先规范项目类别并执行结构性资格门控，再实时核验动态规则，最后依据公开标准对五个适配维度评分。资格、适配分数和证据置信度始终分开。

## 产出

- 主推荐、备选和不建议申报路径。
- 每个奖项的资格状态、动态待核验项和适配解释。
- 申报优先级、证据置信度和下一步资料需求。

## 内置参考

包含 iF、iF Student、Red Dot Product、Red Dot Concept、IDEA、DIA、K-Design、GOOD DESIGN Japan、Core77、James Dyson 与 EPDA 的版本化配置和官方来源。

## 边界

- 不把适配度解释为获奖概率。
- 不默认为缺失资格信息通过；使用 `Unknown`。
- 不负责检索获奖案例、全面设计评价或最终文件合规检查。

## 相关技能
